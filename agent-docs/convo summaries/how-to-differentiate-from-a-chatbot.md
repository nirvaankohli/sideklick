# How to differentiate from a chatbot

Summary:
- Outlined the highest-leverage ways to make the product feel meaningfully different from a generic chat bot.
- Focused on features that fit the current codebase: persistent class memory, gap-aware review, context-sensitive actions, and workflow integration.

How to run:
```powershell
npm run dev
```

Product directions to test:
- Make the assistant proactively use saved gaps and session summaries.
- Turn review and focus actions into memory-driven study flows instead of one-off chat replies.
- Make browser actions and screenshots produce context-aware help tied to the current class.
